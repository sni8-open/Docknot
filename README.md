# DocKnot вЂ“ Local RAG Chat Groups

DocKnot is a **Flask-based web application** that allows users to upload PDFs and ask questions about them using a **Retrieval-Augmented Generation (RAG)** system powered by **Ollama (local LLM)** and **ChromaDB (vector database)**.

The system organizes documents into **chat groups**, allowing topic-based document analysis.

Example:

- Group: Statistics
- Upload statistics PDFs
- Ask questions related to those PDFs

The system retrieves relevant document chunks and generates answers with **citations**.

---

# Features

- Upload PDFs into topic-based groups
- Ask questions about uploaded documents
- Retrieval-Augmented Generation (RAG)
- Local LLM using **Ollama**
- Local embeddings using **nomic-embed-text**
- Vector search using **ChromaDB**
- Streaming responses (typewriter style)
- Source citations (PDF + chunk)
- Delete PDF and remove vectors
- Runs completely **offline**

# Prerequisites

Before running the project install:

- Python **3.12**
- Ollama

Download Ollama:
https://ollama.com/download

---

# Setup Ollama Models

Pull the required models:
ollama pull llama3.1
ollama pull nomic-embed-text

These models will be used for:

- LLM responses
- Embedding generation

---

# Installation

Clone the repository:
https://github.com/sni8-open/Docknot.git

cd Docknot

Create virtual environment & Activate:
python -m venv venv

Install dependencies:
pip install -r requirements.txt

---

# Initialize Database

Run:
python -m db.init_db

This creates:
app.db

---

# Run the Application

Start Ollama:
ollama serve

Then start Flask:
python app.py

Open browser:
http://127.0.0.1:5000

---

# How to Use

1. Create a **Group** (example: Statistics)
2. Upload PDF documents
3. Ask questions related to those documents
4. The system will:
   - Retrieve relevant chunks
   - Generate answers using LLM
   - Show citations

---

# Example Query

What is linear regression?

Response will include:

- Answer generated by LLM
- Source citation
- PDF chunk reference
